# Constitution for Hackathon II: Evolution of Todo

## 1. Strict Technical Rules

### 1.1. Architecture
- **Phase I:** Python Console App with In-Memory Storage.
- **Future Phases:** Evolve towards a Cloud-Native architecture.

### 1.2. Tech Stack
- **Language:** Python 3.13+
- **Package Manager:** UV

### 1.3. Methodology
- **Development Process:** Spec-Driven Development (SDD).
- **Workflow:** Write specifications before generating or writing code.

### 1.4. Coding Standards
- **Style:** Adhere to PEP8 guidelines.
- **Principles:** Follow Clean Code principles for readability and maintainability.
- **Typing:** Use Type Hinting for all new code.

### 1.5. File Structure
- **Source Code:** All source code must reside in the `src/` directory.
- **Specifications:** All specification documents must be located in the `specs/` directory.
